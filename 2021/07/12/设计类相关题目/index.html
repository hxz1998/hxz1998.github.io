1. LRU缓存

题目来自力扣题库【2136】：LRU缓存

1.1 题目描述

设计和构建一个“最近最少使用”缓存，该缓存会删除最近最少使用的项目。缓存应该从键映射到值（允许你插入和检索特定键对应的值），并在初始化时指定最大容量。当缓存被填满时，它应该删除最近最少使用的项目。

它应该支持以下操作： 获取数据 get 和 写入数据 put 。

获取数据 get(key) - 如果密钥 (key) 存在于缓存中，则获取密钥的值（总是正数），否则返回 -1。写入数据 put(key, value) - 如果密钥不存在，则写入其数据值。当缓存容量达到上限时，它应该在写入新数据之前删除最近最少使用的数据值，从而为新的数据值留出空间。

1.2 示例数据

1.3 解题思路

题目要求达到两个目标：

  1. 键值映射。
  2. 当添加数据但是缓存却满了的时候，把最不常用的数据丢掉。

对于第一个问题，显而易见的要用哈希表来完成键值映射，这样一来查找效率高达 $O(1)$，但是修改顺序的复杂度变成了 $O(n)$。
对于第二个目标，却需要使用链表来调整顺序，以达到 $O(1)$ 的修改效率，但是其查找过程却变成了 $O(n)$。
把前面两种数据结构组合起来，就可以解决该题。

首先抽象出来一个双向链表，该链表用于保存顺序信息，然后使用哈希表来映射从 key 到链表中节点 node。这样便可以在 $O(1)$ 时间复杂度内实现查找功能，也能在 $O(1)$ 时间复杂度内实现修改顺序功能。

2. 实现Trie（前缀树）

题目来自力扣题库【208】：实现前缀树

2.1 题目描述

Trie（发音类似 “try”）或者说 前缀树 是一种树形数据结构，用于高效地存储和检索字符串数据集中的键。这一数据结构有相当多的应用情景，例如自动补完和拼写检查。

请你实现 Trie 类：

  • Trie() 初始化前缀树对象。
  • void insert(String word) 向前缀树中插入字符串 word 。
  • boolean search(String word) 如果字符串 word 在前缀树中，返回 true（即，在检索之前已经插入）；否则，返回 false 。
  • boolean startsWith(String prefix) 如果之前已经插入的字符串 word 的前缀之一为prefix ，返回 true ；否则，返回 false 。

注意：

  • 1 <= word.length, prefix.length <= 2000
  • word 和 prefix 仅由小写英文字母组成
  • insert、search 和 startsWith 调用次数 总计 不超过 $3 * 10^4$ 次

2.2 示例数据

2.3 解题思路

非经典的多叉树题，该数据结构描述了两个语义：

  1. 当前节点是否是终结节点
  2. 当前节点代表的字符（字母映射表），如果next[0] != nullptr，那么说明该存在字母a
    该数据结构满足：一次建树，多次查询，但是由于每一个节点只有一个字符语义，所以是非常耗费空间的。

3. 用队列实现栈

题目来自力扣题库【225】：用队列实现栈

3.1 题目描述

请你仅使用两个队列实现一个后入先出（LIFO）的栈，并支持普通栈的全部四种操作（push、top、pop 和 empty）。

实现 MyStack 类：

  • void push(int x) 将元素 x 压入栈顶。
  • int pop() 移除并返回栈顶元素。
  • int top() 返回栈顶元素。
  • boolean empty() 如果栈是空的，返回 true ；否则，返回 false 。

注意：

  • 你只能使用队列的基本操作 —— 也就是 push to back、peek/pop from front、size 和 is empty 这些操作。

  • 你所使用的语言也许不支持队列。 你可以使用 list （列表）或者 deque（双端队列）来模拟一个队列 , 只要是标准的队列操作即可。

  • 1 <= x <= 9。

  • 最多调用100 次 push、pop、top 和 empty。
  • 每次调用 pop 和 top 都保证栈不为空。

3.3 解题思路

这道题是一个简单题，和使用栈实现队列是一样的道理，其重点都在数据结构的灵活使用上，题目本身很简单。

4. 数据流的中位数

题目来自力扣题库【295】：数据流的中位数

4.1 题目描述

中位数是有序列表中间的数。如果列表长度是偶数，中位数则是中间两个数的平均值。例如，

  • [2,3,4] 的中位数是 3
  • [2,3] 的中位数是 (2 + 3) / 2 = 2.5

设计一个支持以下两种操作的数据结构：

  • void addNum(int num) - 从数据流中添加一个整数到数据结构中。
  • double findMedian() - 返回目前所有元素的中位数。

4.2 示例数据

4.3 解题思路

这道题是一个困难题，其难度主要在怎样实现数据流的操作，因为数据流是一个没有准确大小的数据，因此肯定不可以直接使用排序然后再来找中间值的方法。

这里使用最大最小堆来完成，其中堆可以使用优先级队列来表示，其中最大堆保存较小的一半数据，其中数据是按照从大到小排序完成，堆顶保存这一半中最大的数。而最小堆用来保存较大的一半数据，堆顶保存最小的一个数。

这样一来，每插入一个数，其时间复杂度仅为$O(nlog(n))$，而且每次只需要检查两个堆的最大最小值以及堆本身的大小，就可以找到中位数，时间复杂度为$O(1)$。

5. 序列化二叉树

题目来自力扣题库【2014】：序列化二叉树

5.1 题目描述

请实现两个函数，分别用来序列化和反序列化二叉树。

你需要设计一个算法来实现二叉树的序列化与反序列化。这里不限定你的序列 / 反序列化算法执行逻辑，你只需要保证一个二叉树可以被序列化为一个字符串并且将这个字符串反序列化为原始的树结构。

注意：

  • 树中结点数在范围 [0, 104] 内
  • -1000 <= Node.val <= 1000

5.2 示例数据

5.3 解题思路

对于这道题，可以使用的办法有经典的深度优先遍历以及广度优先遍历，也有比较奇葩的递归下降解码方法，其中广度优先遍历（也就是层序遍历）个人感觉比较直观，也比较方便，唯一不足的可能就是写出来的程序不太雅观，但是能运行就可以了😉。

在序列化过程中，当然是层序遍历树中节点，使用 - 来表示该层中的某一个位置上节点为空节点，使用数字来表示原来节点的值，节点之间的分隔符使用 . 来表示。

那么在反序列化过程中，仍然使用层序遍历来完成，遇到 - 就构建一个空节点放到当前父节点之下，否则的话就把值取出来创建一个节点，挂在父节点下。
